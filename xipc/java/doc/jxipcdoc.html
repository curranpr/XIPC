jxipc - XIPC Interactive Command Processor

Name

jxipc - XIPC Interactive Command Processor (a Java application)

Description

jxipc is a command interpreter that provides the user with interactive access to XIPC's Java API.

Most of the interpreter's commands correspond to XIPC API's, and their arguments are the same, except for necessary adjustments to the interactive environment. To find a full description of a command and its arguments, refer to the description of the corresponding API.

The xipc Interactive Language

Syntax

Each command starts with a command verb, usually an XIPC API name. The command name is followed by the command arguments separated by one or more spaces.

Arguments that consist of a list of values, such as SidList, use a comma as a separator between the values.

Text arguments are entered either as a contiguous string of non-blank characters delimited by spaces or as a string delimited by double quotes.

Conventions Used In This Section

The following conventions are used in the description of jxipc command syntax:

o Text in bold is to be entered as specified;

o Items in italics represents values to be provided by the user;

o Items between brackets '[ ]' imply an optional choice.

 

General Interactive Commands

help - Display List Of Arguments

Syntax

help Command
? Command

Arguments

Command Name of xipc command.

Examples

jxipc> help xipclogin
xipclogin InstanceName UserName

quit - Logout And Quit

Syntax

q[uit]

Arguments

None.

uid - Display Current User Id

Syntax

uid

Arguments

None.

Examples

jxipc> uid
Uid = 11

XIPC Interactive Commands

xipcabort - Abort a User

Syntax

xipcabort UserId

Arguments

UserId User id of user to be aborted

Examples

jxipc> xipcabort 11
RetCode = 0

xipcerror - Display Error Message

Syntax

xipcerror ErrorCode

Arguments

ErrorCode XIPC error code

Examples

jxipc> xipcerror -1003
Configuration capacity limit exceeded

xipclogin - Log Into An Instance

Syntax

xipclogin [InstanceName] [UserName]

Arguments

InstanceName Name of a network instance to log into. Must be an instance network name of the form @hostname:instance .

UserName Name to be assigned to the XIPC user.

Examples

jxipc> xipclogin @Server:Instance1 George
Uid = 11

xipclogout - Log Out Of Instance

Syntax

xipclogout

Arguments

None.

Examples

jxipc> xipclogout
RetCode = 0

SemSys Interactive Commands

semaccess - Access An Existing Semaphore

Syntax

semaccess Name

Arguments

Name Semaphore name

Examples

jxipc> semaccess InitSem
Sid = 7

semacquire - Acquire Resource Semaphore

Syntax

semacquire AcquireType SidList BlockingOpt

Arguments

AcquireType any, all or atomic.

SidList List of Semaphore Ids separated by commas.

BlockingOpt wait or nowait or timeout(<int>).

Examples

jxipc> semacquire any 0,1,3 wait
RetCode = 0 Sid = 1

semcancel - Cancel Blocked Operations

Syntax

semcancel SidList

Arguments

SidList List of Semaphore Ids separated by commas.

Examples

jxipc> semcancel 0
RetCode = 1 Sid = 0

semclear - Clear Event Semaphore

Syntax

semclear SidList

Arguments

SidList List of Semaphore Ids separated by commas.

Examples

jxipc> semclear 0,1
RetCode = 0

semcreate - Create A New Semaphore

Syntax

semcreate Name CreateValue

Arguments

Name Name of new semaphore.

CreateValue For a resource semaphore: A maximum value;
For an event semaphore: Either clear or set.

Examples

jxipc> semcreate Resource 5
Sid = 0

jxipc> semcreate InitSem clear
Sid = 3

semdelete - Delete A Semaphore

Syntax

semdelete Sid

Arguments

Sid Semaphore Id.

Examples

jxipc> semdelete 5
RetCode = 0

semdestroy - Destroy A Semaphore

Syntax

semdestroy Sid

Arguments

Sid Semaphore Id.

Examples

jxipc> semdestroy 5
RetCode = 0

semrelease - Release Resource Semaphore

Syntax

semrelease SidList

Arguments

SidList List of Semaphore Ids separated by commas.

Examples

jxipc> semrelease 9,11
RetCode = 0

semset - Set Event Semaphore

Syntax

semset SidList

Arguments

SidList List of Semaphore Ids separated by commas.

Examples

jxipc> semset 0,1,22

semwait - Wait On Event Semaphore

Syntax

semwait WaitType SidList [Clear,]BlockingOpt

Arguments

WaitType any, all or atomic.

SidList List of Semaphore Ids separated by commas.

BlockingOpt wait or nowait or timeout(<int>).

Examples

QueSys Interactive Commands

queaccess - Access An Existing Queue

Syntax

Name Name of new queue.

LimitMsgs Either maximum number of messages in queue or nolimit.

LimitBytes Either maximum number of text bytes in queue or nolimit.

Examples

jxipc> quecreate ServerQueue nolimit 10000

quedelete - Delete A Queue

quedelete Qid

Arguments

Qid Queue Id.

Examples

jxipc> quedelete 5
RetCode = 0

quedestroy - Destroy A Queue

Syntax

quedestroy Qid

Arguments

Qid Queue Id.

Examples

jxipc> quedestroy 5
RetCode = 0

quereceive - Receive And Read A Message From A Queue

Syntax

quereceive QueSelectCode QidList BlockingOpt

Arguments

QueSelectCode A message retrieval queue select code. (This is presented in the QueSys documentation.) The prefix "QUE_Q_" of the queue select code should be omitted, e.g., instead of QUE_Q_EA, use ea.

QidList A list of queue Ids, possibly specified with message select codes. (This is presented in the QueSys documenation.) The prefix "QUE_M_" of the message select code should be omitted, e.g., instead of QUE_M_PRGT(2,100), use prgt(2,100).

BlockingOpt wait or nowait or timeout(<int>).

Examples

jxipc> quereceive ea 0 wait
Qid = 0 Priority = 120 Length = 22
Text = "Mary had a little lamb"

jxipc> quereceive hpq prgt(0,1000),prgt(1,900) nowait
Qid = 1 Priority = 950 Length = 13
Text = "High Priority"

quesend - Write And Send A Message To A Queue

Syntax

quesend QueSelectCode QidList Priority MessageText BlockingOpt

Arguments

QueSelectCode A message dispatch queue select code. (This is presented in the QueSys Functions documentation.) The prefix "QUE_Q_" of the queue select code should be omitted, e.g., instead of QUE_Q_SHQ, use shq.

QidList A list of queue Ids

Priority The priority to be assigned to the message.

MessageText The text of the message enclosed in double quotes.

BlockingOpt wait or nowait or timeout(<int>).

Examples

jxipc> quesend shq 0,1 100 "Mary had a little lamb" wait
RetCode = 0 Qid = 0

MemSys Interactive Commands

memaccess - Access An Existing Segment

Syntax

memaccess Name

Arguments

Name Segment name

Examples

jxipc> memaccess TrackTable
Mid = 1

memcreate - Create A New Memory Segment

Syntax

memcreate Name Size

Arguments

Name Segment name

Size Segment size

Examples

jxipc> memcreate TrackTable 10240
Mid = 1

memdelete - Delete A Memory Segment

Syntax

memdelete Mid

Arguments

Mid Segment Id

Examples

jxipc> memdelete 1
RetCode = 0

memdestroy - Destroy A Memory Segment

Syntax

memdestroy Mid

Arguments

Mid Segment Id

Examples

jxipc> memdestroy 1
RetCode = 0

memlock - Lock Memory Section(s)

Syntax

memlock LockType MidList BlockingOpt

Arguments

LockType any, all or atomic.

MidList List of section variables or memory section descriptors separated by commas. Each descriptor consists of Mid, Offset and Size enclosed in parenthesis.

BlockingOpt wait or nowait or timeout(<int>).

Examples

jxipc> memsection s (1 0 20)
Section = (1 0 20)

jxipc> memlock atomic (0 120 30),(0 4000 30),s wait
RetCode = 0
Section = (1 0 20)

memread - Read Data From A Memory Segment

Syntax

memread Mid Offset Length BlockingOpt

Arguments

Mid Segment Id.

Offset Number of bytes beyond the start of the segment.

Length Number of bytes

BlockingOptwait or nowait or timeout(<int>).

Examples

jxipc> memread 1 0 25 wait
Text = "Larry had a little mutton"

memsection - Set A Section Variable

Syntax

memsection SectionId [Section]

Arguments

SectionId A section variable beginning with a letter a through z.

Section Either a section variable or section descriptor: Mid, Offset and Size enclosed in parenthesis.

Examples

jxipc> memsection sec1 (0 0 100)
Section = (0 0 100)

jxipc> section sec1
Section = (0 0 100)

memunlock - Unlock Memory Section(s)

Syntax

memunlock MidList

Arguments

MidList List of section variables or memory section descriptors separated by commas. Each descriptor consists of Mid, Offset and Size enclosed in parenthesis.

Examples

jxipc> memunlock (0 120 30),(0 4000 30)
RetCode = 0

memwrite - Write Data Into A Memory Segment

Syntax

memwrite Mid Offset Length Text BlockingOpt

Arguments

Mid Segment Id.

Offset Number of bytes beyond the start of the segment.

Length Number of bytes

Text The text to be written enclosed in double quotes or a filler character enclosed in single quotes

BlockingOpt wait or nowait or timeout(<int>).

Examples

jxipc> memwrite 1 0 22 "Mary had a little lamb" wait
RetCode = 0

jxipc> memwrite 1 0 22 ' ' wait
RetCode = 0

section - Display A Section Variable

Syntax

section SectionId

Arguments

SectionId A section variable.

Examples

jxipc> memsection s (0 0 100)
Section = (0 0 100)

jxipc> section s
Section = (0 0 100)
